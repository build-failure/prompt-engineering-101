# Getting Started with Prompt Engineering 101

Welcome! This guide will help you set up and begin your prompt engineering learning journey.

## What You Need

### Required
- **Google Gemini Access**: Free or paid tier
  - Visit [gemini.google.com](https://gemini.google.com) to sign up
  - Works in any modern web browser
- **Google Workspace Account**: For integration examples
  - Gmail, Google Docs, Sheets, Slides, Meet
  - Free personal account works fine

### Optional but Helpful
- **Text Editor**: For viewing markdown files locally
  - Any text editor works (Notepad, TextEdit, VS Code, etc.)
  - Or view directly on GitHub/GitLab
- **Note-Taking Tool**: To track your progress and insights

## How to Use This Course

### 1. Navigation

Each chapter has two files:
- **lesson.md** — Concepts and explanations
- **lab.md** — Hands-on exercises

Navigate using the links at the top and bottom of each page:
```
← Previous | Home | Next →
```

### 2. Learning Flow

**For each chapter:**

1. **Read the lesson** (10-20 minutes)
   - Understand the concepts
   - Review the examples
   - Note key techniques

2. **Complete the lab** (15-20 minutes)
   - Follow step-by-step instructions
   - Copy prompts into Gemini
   - Compare your results with expected outputs

3. **Validate your work** (5 minutes)
   - Use the validation prompts provided
   - Self-assess your understanding
   - Identify areas for improvement

4. **Practice with real tasks** (ongoing)
   - Apply techniques to your actual work
   - Adapt templates to your needs
   - Build your prompt library

### 3. Working with Prompts

**How to use the prompt templates:**

1. Find a prompt template in a lab exercise
2. Copy the entire prompt
3. Customize the bracketed sections [like this]
4. Paste into Gemini
5. Review the output
6. Iterate if needed

**Example:**
```
Template:
"Summarize this [document type] for [audience]. Focus on [key aspects]."

Your version:
"Summarize this quarterly report for executives. Focus on revenue trends and key risks."
```

### 4. Self-Assessment

Each lab includes **validation prompts** to help you check your work:

1. Complete an exercise
2. Copy the validation prompt
3. Paste it into Gemini along with your prompt and output
4. Review Gemini's feedback
5. Refine your approach based on suggestions

**Example validation prompt:**
```
I'm learning prompt engineering. Please review my work:

My Prompt: [paste your prompt]
Output I Received: [paste Gemini's response]

Please assess:
1. Is my prompt clear and specific?
2. Does the output meet the goal?
3. What could I improve?
```

## Common Questions

### Q: Do I need programming experience?
**A:** No! This course is designed for non-technical users. Everything is explained in plain language.

### Q: How long does it take to complete?
**A:** 5-6 hours total, but you can work at your own pace. Many learners spread it over 1-2 weeks.

### Q: Can I skip chapters?
**A:** Not recommended. Each chapter builds on previous concepts. Start with Chapter 1 even if you have some experience.

### Q: What if I get stuck?
**A:** Check the [Troubleshooting Guide](resources/troubleshooting-guide.md) or review the relevant lesson. The validation prompts can also help identify issues.

### Q: Can I use this with other AI tools?
**A:** The course is designed for Google Gemini, but many techniques apply to other AI tools. Some syntax may differ.

### Q: Do I need the paid version of Gemini?
**A:** No, the free tier works fine for learning. Paid tiers offer faster responses and higher limits.
