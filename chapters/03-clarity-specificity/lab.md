## Exercise 1: Vague to Specific Transformation

**Objective**: Practice identifying and fixing vague language

**Scenario**: You have several vague prompts that need improvement.

### Your Task

Transform these vague prompts into specific ones:

**Vague Prompt 1:**
```
Analyze the data
```

**Vague Prompt 2:**
```
Write a good email
```

**Vague Prompt 3:**
```
Summarize this briefly
```

### Transformation Template

For each prompt, add:
- What data/content specifically?
- What type of analysis/writing/summary?
- For whom (audience)?
- What format?
- How long?
- What focus?

### Example Solutions

<details>
<summary>Click to see example transformations</summary>

**Prompt 1 Improved:**
```
Analyze this Q1 2026 sales data for the North American region.
Identify: top 3 performing products, any declining categories, and regional trends.
Present as: executive summary paragraph + 3 bullet points with specific percentages.
Audience: Sales leadership team.
```

**Prompt 2 Improved:**
```
Write a professional follow-up email to Jennifer Martinez (procurement manager at TechCorp)
three days after our product demo. Thank her for attending, reference her specific
interest in our automation features, and propose a 30-minute call next Tuesday to
discuss pricing for 100 users. Tone: professional but warm. Length: 3 paragraphs.
```

**Prompt 3 Improved:**
```
Summarize this 15-page market research report in exactly 5 bullet points,
maximum 30 words each. Focus only on: target customer demographics, main
competitors, and market opportunities. Skip methodology and data sources.
Audience: Marketing team planning Q2 campaign.
```

</details>

```

4. Is it too rigid or just right?
```

### Success Criteria
- [ ] Eliminated all vague words (some, good, brief, etc.)
- [ ] Added specific quantities and measurements
- [ ] Defined audience and purpose

## Exercise 2: Apply the 5 W's Technique

**Objective**: Use Who, What, When, Where, Why to add specificity

**Scenario**: You need to create a project status update.

### Your Task

Build a specific prompt using the 5 W's:

**Basic prompt:**
```
Create a project status update
```

**Apply 5 W's:**

**Who:**
- Who is this for? [Your answer]
- Who is involved in the project? [Your answer]

**What:**
- What project specifically? [Your answer]
- What information should be included? [Your answer]
- What format? [Your answer]

**When:**
- What time period does this cover? [Your answer]
- When is the deadline? [Your answer]

**Where:**
- Where will this be shared? [Your answer]
- What platform/medium? [Your answer]

**Why:**
- Why is this update needed? [Your answer]
- What decision does it support? [Your answer]

### Build Your Prompt

Combine your 5 W's answers into a complete prompt:

```
[Your complete prompt here]
```

### Example Solution

<details>
<summary>Click to see example</summary>

```
Create a weekly project status update for the website redesign project.

WHO: For my manager and the executive sponsor
WHAT: Project status covering progress, blockers, and next steps
WHEN: Week of Feb 19-23, 2026
WHERE: Email format for distribution to stakeholders
WHY: Weekly check-in to ensure project stays on track and flag any issues

Include:
- Overall status (on track/at risk/behind)
- This week's accomplishments (3-5 bullets)
- Next week's planned work (3-5 bullets)
- Current blockers or risks (if any)
- Budget status (spent vs. allocated)

Format: Professional email, maximum 250 words
Tone: Factual and clear, flag concerns directly
```

</details>

### Validation Prompt

```
I used the 5 W's technique to create this prompt:

[Paste your prompt]

Please assess:
1. Did I address all 5 W's?
2. Is the prompt specific and actionable?
3. Would this produce consistent, useful results?
4. What's missing or could be improved?
```

### Success Criteria
- [ ] Addressed all 5 W's (Who, What, When, Where, Why)
- [ ] Each W adds meaningful specificity
- [ ] Prompt is clear and actionable
- [ ] Would produce consistent results

## Exercise 3: Data Analysis with Specificity

**Objective**: Write a specific prompt for analyzing business data

**Scenario**: You have monthly sales data and need insights for a team meeting.

### Your Task

Write a specific data analysis prompt that includes:

**Required elements:**
- Exact data source and time period
- Specific metrics to analyze
- Type of analysis needed
- Format for output
- Audience and purpose
- What to focus on
- What to exclude

### Prompt Template

```
Analyze [SPECIFIC DATA] from [TIME PERIOD].

Focus on:
- [METRIC 1]
- [METRIC 2]
- [METRIC 3]

Identify:
- [WHAT TO FIND 1]
- [WHAT TO FIND 2]
- [WHAT TO FIND 3]

Format:
- [STRUCTURE]

Audience: [WHO]
Purpose: [WHY]
Exclude: [WHAT NOT TO INCLUDE]
```

### Example Scenario

**Data available:**
- Monthly sales by product category (Jan-Mar 2026)
- Revenue, units sold, average order value
- Regional breakdown (North, South, East, West)

**Meeting context:**
- Sales team meeting next Tuesday
- Need to identify what's working and what needs attention
- 10 minutes for presentation

### Write Your Prompt

```
[Your specific data analysis prompt here]
```

### Example Solution

<details>
<summary>Click to see example</summary>

```
Analyze Q1 2026 monthly sales data (January through March) by product category and region.

Data includes:
- Revenue per category per month
- Units sold per category per month
- Average order value per category
- Regional breakdown (North, South, East, West)

Focus on:
- Which product categories grew or declined
- Which regions are outperforming or underperforming
- Any notable trends or patterns

Identify:
- Top 3 performing categories (by revenue growth %)
- Bottom 2 categories needing attention
- Regional performance differences
- Any month-over-month trends

Format:
- Executive summary (2-3 sentences)
- Key findings (5-7 bullet points with specific percentages)
- Top 3 recommendations (actionable, specific)

Audience: Sales team (mix of reps and managers)
Purpose: Inform Q2 strategy and identify focus areas
Exclude: Detailed methodology, raw data tables, statistical jargon

Keep insights practical and action-oriented.
```

</details>

### Validation Prompt

```
I created a data analysis prompt. Please review:

[Paste your prompt]

Please assess:
1. Is the data source and timeframe specific?
2. Are the analysis requirements clear?
3. Is the output format well-defined?
4. Would an analyst know exactly what to do?
5. What would improve this prompt?
```

### Success Criteria
- [ ] Specific data source and time period
- [ ] Clear metrics and analysis focus
- [ ] Defined output format
- [ ] Audience and purpose stated
- [ ] Exclusions noted
- [ ] Actionable and practical

## Exercise 4: Customer Email with Precision

**Objective**: Write a specific prompt for customer communication

**Scenario**: A customer complained about a delayed shipment. You need to draft a response.

### Your Task

Write a specific prompt that includes:

**Customer context:**
- Customer name: Alex Thompson
- Issue: Order #12847 delayed by 5 days
- Customer tone: Frustrated but polite
- Order: Premium subscription renewal ($299)
- Company policy: Offer 15% discount for delays over 3 days

**Your prompt should specify:**
- Exact situation and context
- Tone and approach
- What to include in response
- What to offer
- Length and structure
- Call-to-action

### Write Your Prompt

```
[Your specific customer email prompt here]
```

### Example Solution

<details>
<summary>Click to see example</summary>

```
Draft a customer service response email to Alex Thompson regarding delayed order #12847.

Situation:
- Order: Premium subscription renewal ($299)
- Delay: 5 days past promised delivery
- Customer tone: Frustrated but polite in their complaint
- Company policy: 15% discount for delays over 3 days

Email must include:
1. Sincere apology acknowledging the specific delay (5 days)
2. Brief explanation (shipping partner issue, already resolved)
3. Current status (shipped yesterday, tracking number: [placeholder])
4. Compensation: 15% discount ($45) on next renewal
5. Direct contact info for any questions

Tone: Empathetic and solution-focused, taking full responsibility
Structure:
- Apologize and acknowledge (1 paragraph)
- Explain and provide update (1 paragraph)
- Offer compensation and next steps (1 paragraph)
- Close with contact info

Length: 3 paragraphs, approximately 150 words
Avoid: Over-explaining, making excuses, corporate jargon

Goal: Restore customer confidence and satisfaction
```

</details>

## Exercise 5: Balancing Specificity

**Objective**: Find the sweet spot between too vague and too rigid

**Scenario**: You need to create product descriptions for your e-commerce site.

### Your Task

Review these three versions and identify which is best:

**Version A (Too Vague):**
```
Write a product description
```

**Version B (Just Right):**
```
Write a product description for [product name] targeting [audience].

Include:
- Opening hook (1 sentence highlighting main benefit)
- Key features (3-4 bullets, each starting with a benefit)
- Who it's for (1 sentence)
- Call-to-action

Length: 100-150 words
Tone: Enthusiastic but not salesy, focus on benefits over features
```

**Version C (Too Rigid):**
```
Write a product description with exactly 127 words. First sentence must be
exactly 12 words and start with "Discover". Second sentence must be 15 words
and include the word "innovative". Use exactly 4 bullet points, each exactly
8 words, starting with these specific words: "Enables", "Provides", "Delivers",
"Ensures". Final sentence must be exactly 10 words and end with exclamation point.
```

### Questions

1. Why is Version A too vague?
2. Why is Version B appropriate?
3. Why is Version C too rigid?
4. When might you need more specificity than Version B?
5. When might you need less?

## Key Takeaways

- **Vague language produces vague results** - Be specific about what you want
- **The 5 W's provide structure** - Who, What, When, Where, Why
- **Specificity improves consistency** - Same prompt, similar results
- **Balance is important** - Specific enough to guide, flexible enough for quality
- **Replace vague terms** - Use exact numbers, clear descriptions, defined criteria
