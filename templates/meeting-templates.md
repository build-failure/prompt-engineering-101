# Meeting Prompt Templates

Templates for meeting preparation, facilitation, and follow-up.

---

## Pre-Meeting Templates

### Meeting Agenda Creation
```
Create a meeting agenda for [MEETING TYPE].

Meeting details:
- Purpose: [MAIN OBJECTIVE]
- Duration: [MINUTES]
- Attendees: [LIST PARTICIPANTS AND ROLES]
- Date/Time: [WHEN]
- Location: [IN-PERSON/VIRTUAL/HYBRID]

Objectives:
- [OBJECTIVE 1]
- [OBJECTIVE 2]
- [OBJECTIVE 3]

Include:
- Meeting header (title, date, time, location, attendees)
- Agenda items with time allocations
- Item owners (who leads each discussion)
- Pre-meeting preparation (what to review)
- Expected outcomes

Format: Professional agenda suitable for distribution
Total time must equal: [DURATION]
Include: Buffer time for overruns and breaks if needed
```

### Pre-Meeting Brief
```
Create a pre-meeting brief for attendees.

Meeting: [TITLE]
Date: [WHEN]
Duration: [TIME]
Attendees: [WHO]

Include:
- Meeting purpose and goals
- Background context (why we're meeting)
- Key topics to be discussed
- Decisions that need to be made
- Preparation required:
  - [DOCUMENT TO REVIEW]
  - [DATA TO ANALYZE]
  - [QUESTIONS TO CONSIDER]
- What to bring/have ready

Format: Email or document
Tone: Clear and helpful
Length: 1-2 pages maximum
```

### Discussion Questions
```
Generate discussion questions for a meeting about [TOPIC].

Context: [BACKGROUND INFORMATION]
Meeting goal: [WHAT WE WANT TO ACCOMPLISH]
Attendees: [PARTICIPANTS AND PERSPECTIVES]
Time available: [MINUTES FOR DISCUSSION]

Create [NUMBER] questions that:
- Are open-ended (not yes/no)
- Encourage diverse perspectives
- Build on each other logically
- Lead toward [DESIRED OUTCOME]
- Can be discussed in available time

Format: Numbered list with brief context for each question
```

### Participant Briefing
```
Create individual briefings for meeting participants.

Meeting: [TITLE AND PURPOSE]
Participant: [NAME AND ROLE]

Their specific role in meeting:
- [WHAT THEY'LL PRESENT/DISCUSS]
- [DECISIONS THEY NEED TO MAKE]
- [EXPERTISE THEY BRING]

Include:
- Meeting overview
- Their specific responsibilities
- Preparation needed
- Time allocated for their part
- Key questions they should be ready to answer

Tone: Personalized and clear
Format: Brief memo or email
```

## During Meeting Templates

### Meeting Notes Structure
```
Create a meeting notes template for [MEETING TYPE].

Include sections for:
- Meeting details (date, time, attendees, absent)
- Agenda items discussed
- Key discussion points
- Decisions made (with rationale)
- Action items (owner, deadline, status)
- Parking lot (items to revisit)
- Next meeting (date, focus)

Format: Structured template for real-time note-taking
Style: Easy to fill in during meeting
Include: Prompts for what to capture in each section
```

### Real-Time Summary
```
Summarize these meeting notes taken in real-time:

Raw notes:
"""
[PASTE NOTES]
"""

Meeting context: [TOPIC AND PURPOSE]

Organize into:
- Key discussion points (main themes)
- Decisions made (what was decided and why)
- Action items (who, what, when)
- Open questions (what needs follow-up)

Format: Clean, structured summary
Remove: Redundancy and tangents
Keep: Important context and rationale
```

### Decision Documentation
```
Document this meeting decision:

Decision: [WHAT WAS DECIDED]
Context: [SITUATION REQUIRING DECISION]
Options considered: [ALTERNATIVES DISCUSSED]
Rationale: [WHY THIS OPTION CHOSEN]
Participants: [WHO WAS INVOLVED]
Date: [WHEN]

Include:
- Clear statement of decision
- Key factors in the decision
- Expected impact
- Implementation steps
- Review date (if applicable)

Format: Decision record
Tone: Clear and factual
```

## Post-Meeting Templates

### Meeting Summary
```
Create a meeting summary for distribution.

Meeting: [TITLE]
Date: [WHEN]
Attendees: [WHO ATTENDED]

Raw notes or recording: [PASTE OR DESCRIBE]

Summarize:
- Purpose and objectives
- Key discussion points (brief)
- Decisions made (with context)
- Action items (owner, task, deadline)
- Next steps
- Next meeting (if scheduled)

Audience: [ATTENDEES/BROADER TEAM/STAKEHOLDERS]
Format: Email or document
Length: Maximum [WORD COUNT]
Tone: Professional and clear
```

### Action Items Extraction
```
Extract action items from these meeting notes:

Notes:
"""
[PASTE NOTES]
"""

For each action item, identify:
- Task description (specific and clear)
- Owner (who's responsible)
- Deadline (when it's due)
- Dependencies (if any)
- Priority (high/medium/low)
- Success criteria (how we'll know it's done)

Format: Table or structured list
Sort by: [DEADLINE/PRIORITY/OWNER]
Highlight: Urgent or blocking items
```

### Follow-Up Email
```
Draft a follow-up email after [MEETING TYPE].

Meeting: [TITLE AND DATE]
Attendees: [WHO WAS THERE]
Key outcomes: [BRIEF SUMMARY]

Email should:
- Thank participants
- Summarize key decisions
- List action items with owners and deadlines
- Highlight any urgent items
- Confirm next meeting (if applicable)
- Provide contact for questions

Tone: Professional and appreciative
Format: Clear sections with headers
Length: Concise - maximum 300 words
Include: Attachments note if relevant
```

### Meeting Minutes (Formal)
```
Create formal meeting minutes.

Meeting: [OFFICIAL NAME]
Date: [FULL DATE]
Time: [START - END]
Location: [WHERE]
Attendees: [NAMES AND TITLES]
Absent: [WHO WASN'T THERE]
Chair: [WHO LED]
Secretary: [WHO TOOK NOTES]

From these notes: [PASTE NOTES]

Format as formal minutes:
1. Call to order
2. Approval of previous minutes
3. Agenda items (with discussions and decisions)
4. New business
5. Announcements
6. Next meeting
7. Adjournment

Style: Formal, third-person, factual
Include: Motion/second/vote format for decisions
```

## Specialized Meeting Types

### Project Kickoff Meeting
```
Create materials for a project kickoff meeting.

Project: [NAME]
Duration: [MINUTES]
Attendees: [TEAM AND STAKEHOLDERS]

Create:
1. Agenda covering:
   - Introductions
   - Project overview and goals
   - Scope and deliverables
   - Timeline and milestones
   - Roles and responsibilities
   - Communication plan
   - Next steps
   - Q&A

2. Presentation outline (key slides)

3. Discussion topics

Format: Complete meeting package
Include: Time allocations for each section
```

### Retrospective Meeting
```
Create a retrospective meeting structure for [PROJECT/SPRINT/PERIOD].

Duration: [MINUTES]
Participants: [TEAM MEMBERS]

Structure:
- Set the stage (5 min)
- Gather data: What happened? (15 min)
- Generate insights: Why did it happen? (20 min)
- Decide what to do: Actions (15 min)
- Close (5 min)

Include:
- Discussion prompts for each phase
- Activities or exercises
- How to capture insights
- Action item template

Focus: [WHAT WENT WELL / WHAT TO IMPROVE / LESSONS LEARNED]
Tone: Constructive and forward-looking
```

### Brainstorming Session
```
Create a structure for a brainstorming session on [TOPIC].

Challenge: [PROBLEM TO SOLVE]
Duration: [MINUTES]
Participants: [WHO]

Include:
- Problem statement (clear framing)
- Ground rules (no judgment, build on ideas, etc.)
- Warm-up activity
- Brainstorming phases:
  - Divergent thinking (generate many ideas)
  - Convergent thinking (evaluate and select)
- Idea capture method
- Evaluation criteria
- Next steps

Format: Facilitator guide
Include: Time allocations and transitions
```

### One-on-One Meeting
```
Create a structure for a one-on-one meeting.

Participants: [MANAGER AND TEAM MEMBER]
Frequency: [WEEKLY/BIWEEKLY/MONTHLY]
Duration: [MINUTES]

Agenda template:
- Check-in (how are things going?)
- Progress updates (current work)
- Challenges and blockers
- Development and growth
- Feedback (both directions)
- Action items and next steps

Include:
- Discussion prompts for each section
- Time allocations
- Space for notes
- Follow-up tracking

Style: Conversational structure, not rigid
Focus: [PERFORMANCE/DEVELOPMENT/SUPPORT]
```

### Client Meeting
```
Create materials for a client meeting.

Client: [COMPANY NAME]
Purpose: [MEETING OBJECTIVE]
Duration: [MINUTES]
Our attendees: [NAMES AND ROLES]
Client attendees: [NAMES AND ROLES]

Create:
1. Meeting agenda (professional format)
2. Discussion topics and questions
3. Presentation outline (if presenting)
4. Anticipated questions and answers
5. Desired outcomes

Tone: Professional and client-focused
Include: Contingency topics if meeting runs long/short
```

## Meeting Analysis Templates

### Meeting Effectiveness Review
```
Analyze the effectiveness of this meeting:

Meeting type: [TYPE]
Duration: [ACTUAL TIME]
Attendees: [NUMBER AND ROLES]
Stated objectives: [GOALS]
Outcomes: [WHAT HAPPENED]

Evaluate:
- Were objectives achieved?
- Was time used effectively?
- Were the right people there?
- Were decisions made?
- Were action items clear?
- What could be improved?

Provide:
- Effectiveness rating (1-10)
- Strengths
- Areas for improvement
- Recommendations for next time

Format: Brief assessment
```

### Meeting Cost Analysis
```
Calculate the cost and value of this meeting.

Meeting details:
- Duration: [HOURS]
- Attendees: [NUMBER]
- Average hourly rate: [ESTIMATE]
- Preparation time: [HOURS]

Calculate:
- Total cost (time × people × rate)
- Value delivered (decisions, alignment, progress)
- Cost-benefit assessment
- Recommendations (continue/modify/eliminate)

Provide: Analysis of whether meeting was worthwhile
```

---

## Tips for Meeting Prompts

1. **Specify meeting type** - Different meetings need different structures
2. **Include time constraints** - Helps AI allocate time realistically
3. **List participants** - Affects formality and content
4. **State clear objectives** - What should the meeting accomplish?
5. **Request specific formats** - Agenda, notes, summary, etc.
6. **Consider audience** - Who will see these materials?

## Validation Prompt

```
Please review these meeting materials:

**Meeting Type**: [TYPE]
**Purpose**: [OBJECTIVE]
**Duration**: [TIME]
**Attendees**: [WHO]

**Materials**:
[Paste agenda/notes/summary]

**Please assess**:
1. Is the structure appropriate for this meeting type?
2. Are time allocations realistic?
3. Is the content clear and actionable?
4. Is anything important missing?
5. Is the format professional and easy to follow?
6. What would improve these materials?
```
