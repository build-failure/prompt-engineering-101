# Report and Document Prompt Templates

Reusable templates for summarizing, analyzing, and creating business documents.

---

## Summarization Templates

### Executive Summary from Long Report
```
Summarize this [REPORT TYPE] for [AUDIENCE - executives/board/stakeholders].

Focus on:
- [KEY ASPECT 1 - e.g., financial performance]
- [KEY ASPECT 2 - e.g., strategic initiatives]
- [KEY ASPECT 3 - e.g., risks and opportunities]

Ignore: [SECTIONS TO SKIP - e.g., methodology, appendices]

Format:
- Opening paragraph (context and overview)
- Key findings (5-7 bullet points with specific data)
- Recommendations (3-5 actionable items)

Length: Maximum [WORD COUNT]
Tone: Professional, data-driven, suitable for [DECISION TYPE]

Document:
"""
[PASTE DOCUMENT HERE]
"""
```

### Meeting Notes Summary
```
Summarize these meeting notes for [AUDIENCE - team members/manager/stakeholders].

Extract and organize:
- Decisions made (with rationale)
- Action items (with owner and deadline)
- Open questions (requiring follow-up)
- Next meeting date/time

Format: Structured sections with bullet points
Length: Maximum 300 words
Tone: Clear and actionable

Meeting notes:
"""
[PASTE NOTES HERE]
"""
```

### Research Findings Summary
```
Summarize this research on [TOPIC] for [PURPOSE - decision-making/planning/presentation].

Highlight:
- Main conclusions
- Supporting evidence
- Implications for [OUR SITUATION]
- Recommended actions

Format: 
- Executive summary (1 paragraph)
- Key findings (bullet points)
- Recommendations (numbered list)

Length: [WORD COUNT]
Audience: [TECHNICAL LEVEL]

Research content:
"""
[PASTE RESEARCH HERE]
"""
```

### Project Status Summary
```
Create a project status summary for [PROJECT NAME] covering [TIME PERIOD].

Include:
- Overall status: [On Track/At Risk/Behind]
- Completed milestones: [LIST OR DESCRIBE]
- Current progress: [PERCENTAGE OR DESCRIPTION]
- Upcoming milestones: [NEXT 2-3]
- Blockers/risks: [IF ANY]
- Budget status: [IF RELEVANT]

Format: Structured report with clear sections
Audience: [STAKEHOLDER TYPE]
Tone: Professional and factual
Length: 1-2 pages
```

## Analysis Templates

### Data Analysis Report
```
Analyze this [DATA TYPE] and create a report.

Data context:
- Source: [WHERE DATA CAME FROM]
- Time period: [DATE RANGE]
- Purpose: [WHAT DECISION THIS SUPPORTS]

Analysis required:
1. Identify top [NUMBER] [METRICS - performers/trends/issues]
2. Compare [DIMENSION 1] vs [DIMENSION 2]
3. Flag any [CONCERNS - anomalies/declines/risks]
4. Provide [NUMBER] actionable recommendations

Format:
- Executive summary
- Detailed findings with specific numbers
- Visual suggestions (what charts would help)
- Recommendations with expected impact

Data:
"""
[PASTE DATA HERE]
"""
```

### Competitive Analysis
```
Create a competitive analysis comparing [OUR PRODUCT/COMPANY] with [COMPETITORS].

Compare on:
- [DIMENSION 1 - e.g., pricing]
- [DIMENSION 2 - e.g., features]
- [DIMENSION 3 - e.g., target market]
- [DIMENSION 4 - e.g., strengths/weaknesses]

Format: Comparison table + narrative summary
Include: Our competitive advantages and gaps
Tone: Objective and analytical
Length: 2-3 pages

Information:
"""
[PASTE COMPETITOR INFO HERE]
"""
```

### SWOT Analysis
```
Create a SWOT analysis for [TOPIC - project/product/strategy/decision].

Context: [BACKGROUND INFORMATION]

Analyze:
- Strengths: Internal positive factors
- Weaknesses: Internal limitations
- Opportunities: External favorable conditions
- Threats: External risks

Format: 
- SWOT matrix (4 quadrants)
- 3-5 items per quadrant
- Brief explanation for each item
- Strategic recommendations based on SWOT

Tone: Analytical and strategic
```

## Planning Templates

### Project Proposal
```
Create a project proposal for [PROJECT NAME].

Project details:
- Objective: [WHAT WE WANT TO ACHIEVE]
- Background: [WHY THIS IS NEEDED]
- Scope: [WHAT'S INCLUDED/EXCLUDED]
- Timeline: [DURATION AND KEY MILESTONES]
- Budget: [ESTIMATED COST]
- Team: [RESOURCES NEEDED]
- Success criteria: [HOW WE'LL MEASURE SUCCESS]

Format:
1. Executive Summary
2. Problem Statement
3. Proposed Solution
4. Implementation Plan
5. Budget and Resources
6. Timeline
7. Risk Assessment
8. Expected Outcomes

Audience: [DECISION MAKERS]
Tone: Professional and persuasive
Length: 3-5 pages
```

### Strategic Plan
```
Create a strategic plan for [INITIATIVE/DEPARTMENT/GOAL] for [TIME PERIOD].

Include:
- Vision: [WHERE WE WANT TO BE]
- Current state: [WHERE WE ARE NOW]
- Goals: [3-5 SPECIFIC OBJECTIVES]
- Strategies: [HOW WE'LL ACHIEVE GOALS]
- Key initiatives: [MAJOR PROJECTS/PROGRAMS]
- Metrics: [HOW WE'LL TRACK PROGRESS]
- Resources needed: [BUDGET/PEOPLE/TOOLS]

Format: Structured document with clear sections
Tone: Strategic and forward-looking
Length: [PAGE COUNT]
```

### Action Plan
```
Create an action plan to [ACHIEVE SPECIFIC GOAL].

Goal: [SPECIFIC, MEASURABLE OBJECTIVE]
Deadline: [TARGET DATE]

For each action item, include:
- Task description
- Owner/responsible person
- Deadline
- Dependencies
- Resources needed
- Success criteria

Format: Table or structured list
Priority: [HOW TO PRIORITIZE - by date/impact/dependency]
Include: Risk mitigation for critical items
```

### Roadmap
```
Create a [TIMEFRAME] roadmap for [PRODUCT/PROJECT/INITIATIVE].

Organize by:
- [TIME PERIODS - quarters/months/phases]

For each period, include:
- Major milestones
- Key deliverables
- Dependencies
- Resource requirements

Format: Timeline view with clear phases
Include: Critical path and key decision points
Audience: [STAKEHOLDERS]
```

## Documentation Templates

### Process Documentation
```
Document the process for [PROCESS NAME].

Include:
- Purpose: [WHY THIS PROCESS EXISTS]
- Scope: [WHEN TO USE THIS PROCESS]
- Roles: [WHO'S INVOLVED]
- Steps: [DETAILED PROCEDURE]
- Tools/systems: [WHAT'S NEEDED]
- Exceptions: [SPECIAL CASES]
- Troubleshooting: [COMMON ISSUES]

Format: Step-by-step guide with clear numbering
Audience: [USER TYPE - new employees/team members/all staff]
Tone: Clear and instructional
Include: Screenshots or diagrams if helpful (describe what they should show)
```

### Policy Document
```
Create a policy document for [POLICY TOPIC].

Include:
- Purpose and scope
- Who this applies to
- Policy statement (the rules)
- Procedures (how to comply)
- Responsibilities
- Exceptions
- Consequences of non-compliance
- Effective date
- Review schedule

Format: Formal policy structure
Tone: Clear, authoritative, professional
Length: [PAGE COUNT]
Audience: [ALL EMPLOYEES/SPECIFIC GROUP]
```

### Standard Operating Procedure (SOP)
```
Create an SOP for [TASK/PROCESS].

Structure:
1. Purpose
2. Scope
3. Responsibilities
4. Materials/Equipment needed
5. Procedure (detailed steps)
6. Quality checks
7. Documentation requirements
8. Safety/compliance notes

Format: Numbered steps with sub-steps
Tone: Clear, precise, instructional
Include: Decision points and what to do if issues arise
```

### User Guide
```
Create a user guide for [PRODUCT/FEATURE/SYSTEM].

Audience: [USER TYPE AND TECHNICAL LEVEL]

Include:
- Overview (what it is and why use it)
- Getting started (first-time setup)
- Key features (how to use main functions)
- Common tasks (step-by-step)
- Troubleshooting (FAQs and solutions)
- Tips and best practices

Format: Sections with clear headings, numbered steps
Tone: Friendly and helpful, not overly technical
Length: [PAGE COUNT]
```

## Tips for Using Report Templates

1. **Provide complete context** - AI needs background to create relevant reports
2. **Specify your audience** - Different stakeholders need different detail levels
3. **Include actual data** - Paste real content for AI to work with
4. **Define success criteria** - What makes this report useful?
5. **Request specific formats** - Tables, bullets, sections, etc.
6. **Iterate if needed** - Refine based on first output

## Validation Prompt for Reports

```
Please review this report I generated:

**Report Purpose**: [What this report is meant to accomplish]
**Intended Audience**: [Who will read this]

**Report Content**:
[Paste the generated report]

**Please assess**:
1. Is the structure logical and easy to follow?
2. Are key points clearly highlighted?
3. Is the level of detail appropriate for the audience?
4. Are recommendations actionable?
5. Is the tone professional and appropriate?
6. What could be improved?

Provide specific feedback.
```
