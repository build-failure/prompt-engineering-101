# Presentation Prompt Templates

Templates for creating presentation outlines, slide content, and speaker notes.

---

## Outline Generation

### Topic to Presentation Outline
```
Create a presentation outline on [TOPIC] for [AUDIENCE].

Presentation details:
- Duration: [MINUTES]
- Purpose: [INFORM/PERSUADE/TRAIN/PITCH]
- Audience knowledge level: [BEGINNER/INTERMEDIATE/EXPERT]
- Key messages:
  - [MESSAGE 1]
  - [MESSAGE 2]
  - [MESSAGE 3]

Include:
- Title slide
- Agenda/overview
- Main content sections (with slide count per section)
- Conclusion
- Q&A slide

Format: Outline with slide titles and brief content description
Suggest: Approximate time per section
```

### Document to Presentation
```
Convert this document into a presentation outline.

Document type: [REPORT/PROPOSAL/ARTICLE]
Target presentation length: [MINUTES or SLIDE COUNT]
Audience: [WHO WILL ATTEND]

Extract and organize:
- Main themes (become sections)
- Key data points (become slides)
- Supporting evidence (become bullet points)
- Conclusions (become final slides)

Exclude: [WHAT TO LEAVE OUT - details/methodology/etc.]

Format: Slide-by-slide outline with titles and main points
```

### Data to Story Presentation
```
Create a presentation that tells the story of this data.

Data: [DESCRIBE OR PASTE DATA]
Story angle: [WHAT NARRATIVE - growth/problem/opportunity/comparison]
Audience: [STAKEHOLDERS]
Goal: [WHAT DECISION OR ACTION]

Structure:
- Setup: Context and why this matters
- Data reveal: Key findings with visuals
- Insights: What this means
- Recommendations: What to do next

Slide count: [NUMBER]
Include: Suggestions for chart types and visuals
```

### Pitch Deck Outline
```
Create a pitch deck outline for [PRODUCT/SERVICE/IDEA].

Audience: [INVESTORS/CLIENTS/PARTNERS]
Duration: [MINUTES]

Include these standard sections:
1. Problem (what pain point exists)
2. Solution (our product/service)
3. Market opportunity (size and potential)
4. Product/service details (how it works)
5. Business model (how we make money)
6. Traction (progress to date)
7. Competition (landscape and our advantage)
8. Team (who we are)
9. Financials (projections)
10. Ask (what we need)

For each slide: Title and 2-3 key points
Tone: Compelling and confident
```

## Content Creation

### Slide Content from Outline
```
Create detailed slide content for this presentation outline:

[PASTE OUTLINE]

For each slide, provide:
- Slide title
- Main headline or key message
- Bullet points (3-5 per slide maximum)
- Supporting data or examples
- Visual suggestions (what image/chart would help)

Guidelines:
- Keep text minimal (slides are visual aids, not documents)
- Use action-oriented language
- Include specific numbers and facts
- Suggest where to use visuals vs. text

Audience: [AUDIENCE TYPE]
Tone: [PROFESSIONAL/CASUAL/TECHNICAL]
```

### Individual Slide Content
```
Create content for a slide about [TOPIC].

Purpose: [WHAT THIS SLIDE SHOULD ACCOMPLISH]
Context: [WHERE IT FITS IN PRESENTATION]
Audience: [WHO'S WATCHING]

Include:
- Compelling title (not just topic name)
- Key message (one main point)
- Supporting points (2-4 bullets)
- Data/example if relevant
- Visual suggestion

Keep it: Concise, impactful, easy to read from distance
```

### Data Visualization Slide
```
Create a slide presenting this data:

[PASTE OR DESCRIBE DATA]

Purpose: [WHAT INSIGHT TO HIGHLIGHT]
Audience: [TECHNICAL LEVEL]

Provide:
- Slide title (the insight, not just "Data")
- Chart type recommendation (bar/line/pie/etc.)
- What to emphasize (colors, callouts, annotations)
- Supporting text (1-2 sentences maximum)
- Key takeaway (what audience should remember)

Make it: Clear, focused on one main point
```

## Speaker Notes

### Generate Speaker Notes from Slides
```
Create detailed speaker notes for these slides:

[PASTE SLIDE CONTENT]

For each slide, provide:
- Opening (how to introduce this slide)
- Talking points (what to say about each bullet)
- Transitions (how to move to next slide)
- Timing (approximate minutes per slide)
- Potential questions (and how to answer)

Style: Conversational but professional
Detail level: [BRIEF REMINDERS / FULL SCRIPT]
Total presentation time: [MINUTES]
```

### Speaker Notes for Specific Slide
```
Create speaker notes for this slide:

Slide title: [TITLE]
Slide content:
- [BULLET 1]
- [BULLET 2]
- [BULLET 3]

Context: [WHERE IN PRESENTATION]
Audience: [WHO'S LISTENING]
Time allocation: [MINUTES]

Include:
- How to introduce this slide
- What to say about each point (with examples)
- How to handle likely questions
- Transition to next slide

Tone: [FORMAL/CONVERSATIONAL/TECHNICAL]
```

### Presentation Script
```
Create a full presentation script for this outline:

[PASTE OUTLINE]

Duration: [MINUTES]
Audience: [ATTENDEES]
Delivery style: [FORMAL/CONVERSATIONAL/INTERACTIVE]

For each section:
- Opening remarks
- Detailed talking points
- Transitions between slides
- Where to pause for questions
- Emphasis points (what to stress)
- Timing notes

Include:
- Strong opening (hook the audience)
- Clear transitions
- Engaging examples or stories
- Powerful closing

Format: Time-stamped script
```

## Adaptation Templates

### Technical to Executive Version
```
Adapt this technical presentation for an executive audience.

Original presentation: [PASTE OUTLINE OR CONTENT]
Executive audience: [C-LEVEL/BOARD/SENIOR MANAGEMENT]
Time available: [MINUTES - usually shorter]

Changes needed:
- Remove: Technical jargon and implementation details
- Add: Business impact and ROI focus
- Emphasize: Strategic implications and decisions needed
- Simplify: Complex concepts into key takeaways

Provide: Revised outline with executive-friendly content
```

### Long to Short Version
```
Condense this [LONG DURATION] presentation into [SHORT DURATION].

Original content: [PASTE OUTLINE]

Keep:
- Most critical messages
- Key data points
- Essential recommendations

Remove:
- Supporting details
- Nice-to-have examples
- Deep dives

Provide: Streamlined outline hitting main points only
```

### Detailed to Overview
```
Create a high-level overview version of this detailed presentation.

Detailed version: [PASTE CONTENT]
Overview purpose: [INTRO/TEASER/SUMMARY]
Duration: [MINUTES]

Focus on:
- Big picture themes
- Major conclusions
- Key recommendations

Skip:
- Detailed analysis
- Supporting data
- Process details

Format: Executive summary style
```

### Internal to External Version
```
Adapt this internal presentation for external audience.

Internal version: [PASTE CONTENT]
External audience: [CLIENTS/PARTNERS/PUBLIC]

Changes:
- Remove: Internal jargon, confidential info, internal processes
- Add: Context that external audience needs
- Adjust: Tone to be more formal/polished
- Clarify: Acronyms and company-specific terms

Provide: External-ready version
```

## Special Purpose Templates

### Training Presentation
```
Create a training presentation on [TOPIC].

Audience: [TRAINEES - new employees/team/users]
Duration: [MINUTES]
Learning objectives:
- [OBJECTIVE 1]
- [OBJECTIVE 2]
- [OBJECTIVE 3]

Include:
- Introduction (why this matters)
- Core content (step-by-step or concept-by-concept)
- Examples and scenarios
- Practice/application slides
- Summary and resources
- Q&A

Style: Educational, interactive, clear
Include: Suggestions for activities or discussions
```

### Sales Presentation
```
Create a sales presentation for [PRODUCT/SERVICE].

Prospect: [COMPANY/INDUSTRY]
Their pain points: [KNOWN CHALLENGES]
Duration: [MINUTES]

Structure:
- Opening (establish credibility)
- Their challenge (show you understand)
- Our solution (how we solve it)
- How it works (brief demo or explanation)
- Benefits (specific to their situation)
- Proof (case studies, testimonials)
- Pricing (if appropriate)
- Next steps (clear call-to-action)

Tone: Consultative, not pushy
Focus: Their success, not our features
```

### Conference/Event Presentation
```
Create a conference presentation on [TOPIC].

Event: [CONFERENCE NAME]
Session type: [KEYNOTE/BREAKOUT/PANEL]
Duration: [MINUTES]
Audience: [INDUSTRY PROFESSIONALS/GENERAL/TECHNICAL]

Include:
- Attention-grabbing opening
- Clear thesis or main argument
- Supporting evidence and examples
- Audience engagement points
- Memorable takeaways
- Strong closing

Style: [INSPIRATIONAL/EDUCATIONAL/THOUGHT-PROVOKING]
Include: Suggestions for audience interaction
```

## Tips for Presentation Prompts

1. **Specify slide count** - Helps AI scope the content appropriately
2. **Define audience clearly** - Technical level affects content depth
3. **State the goal** - Inform, persuade, train, or sell?
4. **Request visual suggestions** - AI can recommend chart types and images
5. **Ask for timing** - Get realistic time allocations per section
6. **Include context** - Where and how will this be presented?

## Validation Prompt

```
Please review this presentation content:

**Presentation Purpose**: [GOAL]
**Audience**: [WHO]
**Duration**: [TIME]

**Content**:
[Paste outline or slides]

**Please assess**:
1. Is the structure logical and flows well?
2. Is the content appropriate for the audience?
3. Can this realistically fit in the time allocated?
4. Are key messages clear and memorable?
5. Is there too much or too little content per slide?
6. What would make this presentation more effective?
```
